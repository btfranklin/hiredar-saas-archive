## 2. Source Text ‚Üí Embeddings

### 2.1 Talent Sheet Sections

The following _strings_ are pulled from each published `TalentSheet`, prepended with a section label and embedded with OpenAI:

| Section                       | Raw field                                         | Enriched text sent to OpenAI |
| ----------------------------- | ------------------------------------------------- | ---------------------------- |
| Promotional Blurb             | `promotional_blurb`                               | `"Section: Promotional Blurb | {text}"` |
| Skill Overview                | `skill_overview`                                  | `"Section: Skill Overview | {text}"` |
| Ideal Roles (aspirations)     | `ideal_roles` (comma-separated)                   | `"Section: Ideal Roles | {text}"` |

Vector ID pattern: `talent_{talent_sheet_id}_{section_slug}` where `section_slug` is the lowercase section name with spaces ‚Üí underscores (e.g. `promotional_blurb`).  Vectors live in the Pinecone namespace **`talent_sheets`** with rich metadata (talent id, pool id, preview snippet, etc.).

### 2.2 Job Opening Sections

For every *active* job opening we embed the following:

| Section            | Raw field(s) concatenated                                     | Enriched text |
| ------------------ | ------------------------------------------------------------- | ------------- |
| Job Overview       | `title + "\n" + description`                                | `"Section: Job Overview | {text}"` |
| Required Skills    | `required_skills` (pipe-delimited)                            | `"Section: Required Skills | {text}"` |
| Responsibilities   | `responsibilities + daily_tasks + performance_expectations`   | `"Section: Responsibilities | {text}"` |
| Qualifications     | `required_qualifications`                                     | `"Section: Qualifications | {text}"` |
| Soft Skills        | `soft_skills`                                                 | `"Section: Soft Skills | {text}"` |

Vector ID pattern: `job_{job_id}_{section_slug}` (namespace **`job_openings`**).

## 3. Matching Perspectives

The matcher (`apps/matching/core/matching.py`) queries Pinecone four times per job‚áÑtalent pair:

| Perspective  | Query Vector                                   | Filter (namespace/section)      | Stored as `match_type` |
| ------------ | ---------------------------------------------- | -------------------------------- | ---------------------- |
| Holistic     | Mean of **all three** talent (or job) vectors  | Entire opposite namespace        | `holistic` |
| Skills       | Talent **Skill Overview**                      | Job **Required Skills**          | `skills` |
| Experience   | Talent **Promotional Blurb**                   | Job **Responsibilities**         | `experience` |
| Wildcard     | Talent **Ideal Roles**                         | Job **Job Overview**             | `wildcard` |

Matches returned by Pinecone include a cosine-similarity **score 0-1**.  For each perspective the background task writes/updates one `CandidateMatch` row and copies all four raw scores into the model.  UI methods convert scores to a 1-10 rating.

## 4. Worked Example (current DB snapshot)

Below is a real, lightly-truncated extract captured with a one-liner Django shell on 2025-05-10:

```text
TalentSheet 137
  Promotional Blurb : "Alexis Morrison is a proactive and dynamic administrative professional with a proven track record in optimizing office operations and enhancing productivity‚Ä¶"
  Skill Overview    : "Alexis excels in leveraging her technical expertise in Microsoft Office Suite and Google Workspace to streamline administrative tasks‚Ä¶"
  Ideal Roles       : "Office Manager, Administrative Coordinator, Executive Assistant, Project Administrator, Operations Assistant"

JobOpening 1 ‚Äì Chief Medical Research Scientist
  Required Skills   : "Strategic leadership | Team management | Mentoring | Scientific research project planning and execution | Grant writing‚Ä¶"
```

During matching, four similarity searches are issued.  Suppose Pinecone returns a holistic score of `0.83` for this pair; the stored row would be:

```text
CandidateMatch(job_opening_id=1, talent_sheet_id=137,
               match_type='holistic',
               holistic_score=0.83,
               skills_score=0.76,
               experience_score=0.68,
               wildcard_score=0.54)
```

The recruiter UI rounds these to `/10` ratings (e.g., `0.83 ‚Üí 9/10`).

## 5. Querying & Debugging

* Ad-hoc analysis:
  ```bash
  python manage.py analyze_job_match "Alexis Morrison" "Chief Medical Research Scientist"
  ```
* Programmatic check in the shell:
  ```python
  from apps.matching.models import CandidateMatch
  match = CandidateMatch.objects.filter(job_opening_id=1,
                                        talent_sheet_id=137,
                                        match_type='holistic').first()
  print(match.holistic_score, match.holistic_rating)
  ```

## 6. Updating the Pipeline

* **New Sections?**‚ÄÉAdd them to the `fields` dict in the corresponding task module (`job_opening_tasks.py` or `talent_sheet_tasks.py`) and update this doc.
* **Weighting**‚ÄÉAdjust `average_vectors` or introduce weighting before calculating the holistic vector.
* **Thresholds**‚ÄÉChange score cut-offs in `create_candidate_matches` if you want stricter/looser matching.

## 7. Glossary

| Term | Meaning |
| ---- | ------- |
| **Embedding** | Dense vector representation of text created by OpenAI | 
| **Pinecone Namespace** | Logical bucket inside the index (`job_openings`, `talent_sheets`) |
| **Vector ID** | Predictable identifier (`job_12_job_overview`) that lets us fetch/delete embeddings quickly |
| **Perspective** | One of Holistic, Skills, Experience, Wildcard |
